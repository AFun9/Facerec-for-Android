## 1. 应用简介

本应用是一款**完全本地运行**的人脸注册与识别工具。所有推理均在设备端完成，无需联网，不上传任何图像或生物特征数据。

核心技术栈：

| 组件 | 说明 |
|---|---|
| 人脸检测 | SCRFD-500M（`facedet.onnx`），轻量高速 |
| 人脸特征提取 | FaceNet（`facenet.onnx`），输出 512 维 L2 归一化 embedding |
| 推理引擎 | ONNX Runtime Android 1.18.0，CPU 推理 |
| 摄像头 | CameraX，支持前置/后置摄像头预览与拍照 |

## 3. 构建前准备

### 3.1 开发环境

- **JDK 17**（或更高版本）
- **Android Studio Hedgehog（2023.1.1）** 或更高版本
  - Android SDK Platform 34
  - Android SDK Build-Tools 34.x
- **Gradle**：项目自带 Gradle Wrapper，无需单独安装

验证 Java 版本：
```bash
java -version
# 应输出 17.x 或更高
```

### 3.2 获取模型文件

应用需要两个 ONNX 模型文件，**不含在代码仓库中**，需自行获取：

#### facedet.onnx — 人脸检测模型

来源：[InsightFace buffalo_s 模型包](https://github.com/deepinsight/insightface)

```
buffalo_s/
└── det_500m.onnx   ← 将此文件重命名为 facedet.onnx
```

或直接下载：
```bash
# 方式一：InsightFace 官方
pip install insightface
python -c "import insightface; insightface.model_zoo.get_model('buffalo_s')"
# 模型默认下载到 ~/.insightface/models/buffalo_s/det_500m.onnx

# 方式二：手动从 HuggingFace 下载
# https://huggingface.co/datasets/Gourieff/ReActor/tree/main/models/buffalo_l
```

#### facenet.onnx — 人脸特征提取模型

来源：FaceNet ONNX 导出版本，输入 `[1, 3, 160, 160]`，输出 `[1, 512]`。

可从以下途径获取：
- [nyoki-mtl/keras-facenet](https://github.com/nyoki-mtl/keras-facenet) 自行导出
- [ONNX Model Zoo](https://github.com/onnx/models) 查找 FaceNet

> **注意**：确认模型输入尺寸为 160×160，输出维度为 512。

### 3.3 放置模型文件

将两个模型文件放到指定目录：

```
android-app/
└── app/
    └── src/
        └── main/
            └── assets/
                ├── facedet.onnx    ← 人脸检测模型（约 1.7 MB）
                ├── facenet.onnx    ← 人脸特征模型（约 19 MB）
                └── README.md
```

文件名**必须完全匹配**（区分大小写）：`facedet.onnx`、`facenet.onnx`。

## 4. 编译与安装

### 4.1 用 Android Studio 构建

1. 打开 Android Studio，选择 **File → Open**，选择 `android-app/` 目录
2. 等待 Gradle 同步完成（首次同步会下载依赖，需要网络）
3. 连接 Android 手机并开启 **USB 调试**（开发者选项中开启）
4. 点击工具栏的 **▶ Run** 按钮，选择目标设备
5. 等待编译和安装完成，应用自动启动

### 4.2 用命令行构建

```bash
cd android-app

# 编译 Debug 版本
./gradlew assembleDebug

# APK 输出路径
# app/build/outputs/apk/debug/app-debug.apk

# 编译并直接安装到已连接设备
./gradlew installDebug
```

编译 Release 版本（需要配置签名）：
```bash
./gradlew assembleRelease
```

### 4.3 安装 APK

如果已有编译好的 APK：

```bash
# 通过 adb 安装
adb install app/build/outputs/apk/debug/app-debug.apk

# 覆盖安装（保留数据）
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

也可以将 APK 文件传输到手机后，在文件管理器中直接点击安装（需要在系统设置中允许"安装未知来源应用"）。

## 5. 首次启动

1. **授予相机权限**：首次启动会弹出权限申请对话框，点击"允许"。若拒绝，应用会显示授权按钮，点击后重新申请。

2. **模型加载**：应用启动时会自动将 assets 中的模型文件拷贝到应用私有目录（`/data/data/com.facenet.mobile/files/`），首次启动需要约 1～3 秒。后续启动检测到文件完整则直接加载，无需重复拷贝。

3. **进入主界面**：看到全屏摄像头预览画面，底部有三个按钮（注册、识别、设置），表示应用已就绪。

## 6. 功能说明

### 6.1 主界面

```
┌─────────────────────────────────┐
│                                 │
│      ┌─────────────────┐        │  ← 实时识别结果浮层（顶部居中）
│      │  ✓ 张三 (0.78)  │        │
│      └─────────────────┘        │
│                                 │
│                                 │
│       [ 全屏摄像头预览 ]          │
│                                 │
│       ┌──────┐ 人脸检测框（绿色）  │
│       │      │                  │
│       └──────┘                  │
│                                 │
│  ┌────────┐ ┌────────┐ ┌──────┐ │
│  │  注  册 │ │  识  别 │ │ 设置 │ │  ← 底部功能按钮
│  └────────┘ └────────┘ └──────┘ │
└─────────────────────────────────┘
```

- **绿色矩形框**：实时人脸检测结果，标记出画面中检测到的人脸位置
- **顶部悬浮文字**：实时识别或单次识别的结果

### 6.2 注册人脸

点击底部 **"注  册"** 按钮，弹出注册面板。

**操作步骤：**

1. 在"用户名称"输入框中填写姓名（例如：`张三`）
2. 选择采集张数：
   - **单张**：只采集 1 张，快速注册但鲁棒性较低
   - **推荐3张**（默认）：采集 3 张不同角度/表情，识别效果更稳定
3. 将面部朝向镜头，确保：
   - 光线充足（避免逆光）
   - 人脸在画面中清晰可见
   - 面部占画面面积约 2% 以上
4. 点击 **"拍摄采集"** 按钮，等待状态栏显示"采集成功"
5. 若提示"质量未通过"，根据原因调整（见下表）后重拍
6. 达到目标张数后，点击 **"完成注册"** 保存

**质量门控提示说明：**

| 提示 | 原因 | 解决方法 |
|---|---|---|
| 检测分数过低 | 画面中没有明显人脸 | 正对镜头，面部完整入镜 |
| 人脸过小 | 面部距离摄像头太远 | 靠近摄像头，使面部占据更大面积 |
| 清晰度不足 | 图像模糊 | 保持静止，确保光线充足，避免抖动 |

> **提示**：采集多张时，建议轻微改变头部角度（正面、略左、略右各一张），提升识别的鲁棒性。

### 6.3 单次识别

点击底部 **"识  别"** 按钮（绿色），应用拍摄当前画面并与已注册的所有用户进行比对。

**结果说明：**

- `✓ 张三（0.82）`：识别成功，括号内为相似度得分（0～1，越高越可信）
- `未识别（0.31）`：未找到匹配用户，括号内为最接近的得分

> 识别过程约需 0.5～2 秒，期间"识别"按钮变灰禁用。

### 6.4 实时识别

在**设置面板**中开启"实时识别"开关后，应用会按设定的帧处理间隔自动对摄像头预览帧进行识别，结果实时显示在屏幕顶部。

- 识别结果格式：`✓ 张三 (0.78)`
- 若当前帧无法匹配到已注册用户，结果浮层自动消失
- 实时识别使用并发锁，自动跳过处理能力跟不上的帧，不会堆积任务

> **注意**：实时识别会持续消耗 CPU，建议设置合适的帧间隔（默认 280ms），避免设备过热。

### 6.5 删除用户

在**注册面板**底部会列出所有已注册用户，每位用户右侧有红色"删除"按钮。

1. 打开注册面板
2. 在已注册用户列表中找到要删除的用户
3. 点击其右侧的红色 **"删除"** 按钮
4. 该用户的所有 embedding 数据立即从内存和磁盘中移除

> 删除操作**不可撤销**，如需恢复须重新注册。

### 6.6 设置面板

点击底部 **"设  置"** 按钮（紫色），弹出设置面板。

| 设置项 | 默认值 | 说明 |
|---|---|---|
| **实时识别** | 关闭 | 开启后持续对预览帧进行识别 |
| **识别阈值** | 0.45 | 两个 embedding 的余弦相似度阈值，低于此值视为"未识别" |
| **帧处理间隔** | 280 ms | 实时识别时每隔多少毫秒处理一帧 |

## 7. 参数调优指南

### 识别阈值

阈值范围 0.30～0.80，建议区间 0.40～0.65：

| 阈值 | 效果 |
|---|---|
| < 0.40 | 过于宽松，容易将不同人误识别为同一人（误识率高） |
| 0.40～0.50 | 平衡点，适合光线稳定、角度变化不大的场景 |
| 0.50～0.65 | 较严格，适合高安全性场景，可能偶尔拒绝本人（漏识率略高） |
| > 0.65 | 非常严格，日常使用不推荐 |

**建议**：先用 0.45（默认），若出现误识别则上调；若本人经常被拒则下调。

### 帧处理间隔

| 间隔 | 适用场景 |
|---|---|
| 100～200 ms | 高性能设备，追求极低延迟的实时识别 |
| 200～400 ms | 均衡推荐（默认 280 ms） |
| 400～800 ms | 中低端设备，减少发热和电量消耗 |
| > 800 ms | 仅需偶尔识别，以节能为主 |

### 注册张数

| 张数 | 优缺点 |
|---|---|
| 1 张 | 速度快，但角度变化时识别率下降 |
| 3 张（推荐） | 覆盖正面及小角度偏转，综合效果最好 |

| 文件 | 路径 | 说明 |

| 识别模型缓存 | `/data/data/com.facenet.mobile/files/facenet.onnx` | 从 assets 复制而来，启动时完整性校验 |

**导出 JSON 数据（开发调试用）**：
```bash
adb pull /data/data/com.facenet.mobile/files/facereg/registry.json ./registry.json
```

## 9. 常见问题排查

### Q1：应用启动即崩溃

**检查步骤**：
1. 确认 `app/src/main/assets/` 目录中存在 `facedet.onnx` 和 `facenet.onnx` 两个文件
2. 确认文件名拼写完全正确（全小写）
3. 确认模型文件完整，大小应分别约为 1.7 MB 和 19 MB（具体取决于模型来源）
4. 重新执行 `./gradlew clean assembleDebug` 后重装

### Q2：提示"质量未通过：人脸过小"

- 靠近前置摄像头，使面部占据屏幕面积的 1/5 以上
- 人脸短边像素数约需 > 80 像素

### Q3：提示"质量未通过：清晰度不足"

- 确保光线充足（避免昏暗环境）
- 拍照时保持静止，避免运动模糊
- 擦拭摄像头镜头

### Q4：实时识别结果跳动不稳定

- 上调"识别阈值"（例如从 0.45 调至 0.55）
- 增大"帧处理间隔"（例如从 280 ms 调至 500 ms），给每帧更稳定的处理时间
- 确保注册时采集了 3 张（多角度）

### Q5：相同的人识别相似度偏低（< 0.45）

- 重新注册：删除旧数据，在更好的光线条件下重新采集 3 张
- 确认使用的是与注册时相同或相近的摄像头（前置/后置）
- 若佩戴了眼镜/口罩等遮挡物，注册时也应佩戴相同状态

### Q6：编译时 Gradle 同步失败

- 检查网络连接，`settings.gradle.kts` 中已配置阿里云 Maven 镜像，国内网络应可正常使用
- 手动刷新：Android Studio 菜单 **File → Sync Project with Gradle Files**
- 清理缓存：`./gradlew clean`

### Q7：adb 找不到设备

```bash
# 检查设备连接
adb devices

# 若显示 unauthorized，在手机上确认 USB 调试授权弹窗
# 若设备未出现，检查 USB 线和驱动
```

## 10. 隐私说明

- **所有处理本地完成**：人脸图像及 embedding 特征均不会上传到任何服务器
- **不存储原始图像**：注册过程中拍摄的照片仅用于提取 512 维特征向量，原始图像不落盘
- **数据隔离**：注册数据存储于 Android 应用沙盒，其他应用无法读取
- **卸载即清除**：卸载应用后，所有注册数据随应用数据一并删除
