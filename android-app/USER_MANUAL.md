## 一、快速开始

### 1.1 系统要求

| 项目 | 最低 | 推荐 |
|------|------|------|
| Android | 8.0（API 26） | 10.0 及以上 |
| 架构 | arm64-v8a | arm64-v8a |
| 内存 | 1 GB | 2 GB 及以上 |
| 存储 | 约 60 MB（含模型） | 100 MB 以上 |
| 摄像头 | 前置或后置 | 500 万像素及以上 |

### 1.2 获取并放置模型

应用依赖两个 ONNX 模型，**不随代码仓库提供**，需从 GitHub Release 下载后放入工程。

**1. 下载模型**

从 [Releases](https://github.com/AFun9/Facerec-for-Android/releases) 下载：

| 文件名 | 说明 | 约大小 |
|--------|------|--------|
| [facedet.onnx](https://github.com/AFun9/Facerec-for-Android/releases/download/v1.0.0/facedet.onnx) | 人脸检测（SCRFD-500M） | 2.5 MB |
| [facenet.onnx](https://github.com/AFun9/Facerec-for-Android/releases/download/v1.0.0/facenet.onnx) | 人脸特征（FaceNet） | 23 MB |

**2. 放置到工程**

将下载好的两个文件放入以下目录（文件名必须一致，全小写）：

```
android-app/
└── app/src/main/assets/
    ├── facedet.onnx
    ├── facenet.onnx
    └── README.md
```

### 1.3 编译与安装

**方式一：Android Studio**

1. 用 Android Studio 打开 `android-app/` 目录
2. 等待 Gradle 同步完成
3. 连接手机并开启 USB 调试
4. 点击 **Run (▶)**，选择设备

**方式二：命令行**

```bash
cd android-app
./gradlew assembleDebug          # 生成 APK
./gradlew installDebug           # 编译并安装到已连接设备
```

APK 路径：`app/build/outputs/apk/debug/app-debug.apk`

**已有 APK 时安装：**

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

也可将 APK 拷到手机后，在文件管理器中点击安装（需允许「安装未知来源应用」）。

### 1.4 首次启动

1. **相机权限**：首次启动会请求相机权限，选择「允许」。
2. **模型加载**：首次启动会将 assets 中的模型拷贝到应用私有目录，约 1～3 秒；之后启动若文件完整则直接使用。
3. **主界面**：出现全屏摄像头预览和底部三个按钮（注册、识别、设置）即表示就绪。

## 二、使用指南

### 2.1 主界面

- **全屏预览**：实时摄像头画面
- **绿色框**：检测到的人脸框
- **顶部文字**：单次/实时识别的结果（如 `✓ 张三 (0.78)`）
- **底部三按钮**：注册、识别、设置

```
┌─────────────────────────────────┐
│      ┌─────────────────┐        │  识别结果浮层
│      │  ✓ 张三 (0.78)  │        │
│      └─────────────────┘        │
│                                 │
│       [ 全屏摄像头预览 ]          │
│       ┌──────┐ 人脸框（绿色）     │
│       └──────┘                   │
│  ┌────────┐ ┌────────┐ ┌──────┐ │
│  │  注  册 │ │  识  别 │ │ 设置 │ │
│  └────────┘ └────────┘ └──────┘ │
└─────────────────────────────────┘
```

### 2.2 注册人脸

1. 点击 **注册**，在弹层中输入**用户名称**（如：张三）
2. 选择采集张数：**单张** 或 **推荐3张**（默认，效果更稳）
3. 正对镜头，光线充足、人脸清晰、占画面约 2% 以上
4. 点击 **拍摄采集**，看到「采集成功」后再拍下一张（若选 3 张）
5. 达到目标张数后点击 **完成注册**

**若提示「质量未通过」：**

| 提示 | 处理办法 |
|------|----------|
| 检测分数过低 | 正对镜头，面部完整入镜 |
| 人脸过小 | 靠近摄像头，让人脸占更大面积 |
| 清晰度不足 | 保持静止、光线充足、避免抖动 |

建议多张时略微改变角度（正面、略左、略右），识别更稳。

### 2.3 单次识别

点击 **识别**（绿色按钮），对当前画面拍照并与已注册用户比对。

- **✓ 张三（0.82）**：识别为该用户，括号内为相似度（0～1）
- **未识别（0.31）**：无匹配用户，括号内为最高得分

识别约 0.5～2 秒，期间按钮会禁用。

### 2.4 实时识别

在 **设置** 中打开「实时识别」后，会按设定间隔对预览帧自动识别，结果在顶部实时更新。

- 结果格式：`✓ 张三 (0.78)`；无匹配时浮层会消失
- 会持续占用 CPU，可适当调大「帧处理间隔」减轻发热

### 2.5 删除用户

在 **注册** 弹层底部有已注册用户列表，每行右侧有红色 **删除** 按钮，点击即可移除该用户全部数据。**不可恢复**，需恢复请重新注册。

### 2.6 设置

| 项 | 默认 | 说明 |
|----|------|------|
| 实时识别 | 关 | 开启后持续对预览帧识别 |
| 识别阈值 | 0.45 | 相似度低于此值视为未识别 |
| 帧处理间隔 | 280 ms | 实时识别时每多少毫秒处理一帧 |

## 三、参数与数据

### 3.1 参数调优

**识别阈值（0.30～0.80，建议 0.40～0.65）**

- < 0.40：偏松，易误识别人
- 0.40～0.50：日常推荐
- 0.50～0.65：更严格，漏识略增
- 建议先用 0.45，误识多则调高，本人常被拒则调低

**帧处理间隔**

- 100～200 ms：高性能机、低延迟
- 200～400 ms：均衡（默认 280 ms）
- 400～800 ms：中低端机、省电
- > 800 ms：偶尔识别、节能为主

**注册张数**：1 张快但鲁棒性差；3 张（推荐）综合效果更好。

### 3.2 数据存储

数据均在应用私有目录，其他应用不可见，**卸载即清除**。

| 内容 | 路径 |
|------|------|
| 人脸库（二进制） | `files/facereg/registry.bin` |
| 人脸库（JSON） | `files/facereg/registry.json` |
| 模型缓存 | `files/facedet.onnx`、`files/facenet.onnx` |

调试导出 JSON：`adb pull /data/data/com.facenet.mobile/files/facereg/registry.json ./`

## 四、常见问题

**Q1：启动即崩溃**  
确认 `app/src/main/assets/` 下存在 `facedet.onnx`、`facenet.onnx`，文件名全小写，文件完整（约 2.5 MB、23 MB）。再执行 `./gradlew clean assembleDebug` 后重装。

**Q2：人脸过小**  
让人脸占屏幕约 1/5 以上，短边约 80 像素以上。

**Q3：清晰度不足**  
光线充足、保持静止、擦拭镜头。

**Q4：实时识别结果乱跳**  
适当提高识别阈值、增大帧间隔，注册时用 3 张多角度。

**Q5：本人相似度偏低（< 0.45）**  
在更好光线下重新注册 3 张；注册与识别尽量用同一摄像头、同一佩戴状态（眼镜/口罩等）。

**Q6：Gradle 同步失败**  
检查网络；项目已配阿里云镜像；可尝试 **File → Sync Project with Gradle Files** 或 `./gradlew clean`。

**Q7：adb 找不到设备**  
执行 `adb devices`，手机端确认 USB 调试授权；检查数据线和驱动。

## 五、隐私说明

- 所有人脸图像与特征均在设备本地处理，**不上传**任何服务器
- 注册时只提取 512 维特征并保存，**不保存原始照片**
- 数据存于应用沙盒，其他应用无法访问
- **卸载应用后，所有注册数据一并删除**
