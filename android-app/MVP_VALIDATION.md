# MVP Validation Checklist

## 1) 功能验证

- [ ] 首次启动申请相机权限，拒绝后可再次请求。
- [ ] 注册页输入用户ID后，点击拍摄可累计进度。
- [ ] 1张模式可直接完成注册并落盘。
- [ ] 3张模式达到 3/3 后自动完成注册并落盘。
- [ ] 识别页拍摄后可输出匹配用户与相似度。
- [ ] `filesDir/facereg/` 目录生成：
  - `registry.bin`
  - `registry.json`

## 2) 性能验证（建议中端机）

- [ ] 单次拍照到结果：< 120ms（检测+embedding）
- [ ] 连续点击拍摄无明显UI阻塞
- [ ] 内存稳定，无明显增长（10次以上注册/识别）

## 3) 稳定性验证

- [ ] 切后台再回前台，摄像头可恢复
- [ ] 旋转屏幕后仍可继续拍摄
- [ ] 模型文件缺失时给出明确错误提示

## 4) 阈值建议

- 初始识别阈值：`0.45`
- 注册默认采集：`3 张`（推荐），`1 张`（兼容）
- 质量门控建议：
  - 检测分数 `>= 0.45`
  - 最小脸面积占比 `>= 0.06`
  - 清晰度分数 `>= 12`

## 5) 当前开发环境说明

- 当前仓库环境未预装 `gradle` 命令，无法在该终端直接执行 `assembleDebug`。
- 请在 Android Studio 打开 `android-app/` 后完成同步与真机验证。
